# アプリ共通開発ルール (Development Guidelines)

FamilyApp内に新しくアプリ（ページ）を追加開発する際の指標です。
全体での統一感あるUI/UXを提供し、既存機能（認証や仕組み）を最大限再利用するためのルールを定義します。

## 1. 認証とセキュリティ
- **ログイン必須**: アプリはすべて `onAuthStateChanged` で監視し、未ログインなら `index.html`（トップ・ログイン画面）へリダイレクトします。
- **許可されたユーザーのみ**: `ALLOWED_EMAILS` に定義されたアドレスのみアクセスを許可し、第三者の登録・閲覧をシャットアウトします。

## 2. データベース (Firestore)の運用
- **コレクションの独立**: アプリごとに Firestore のコレクションを分けます（例: `kakei_items`, `sokone_items`）。
- **リアルタイム同期**: 夫婦で同時に操作しても齟齬が出ないよう、極力 `onSnapshot` によるリアルタイムリスナーを使用して画面を再描画します。
- **並び順の管理**: 位置関係を保持するための `order`（Number型）フィールドなどを持たせ、順序同期に活用します。

## 3. デザインと操作の統一感
- **トップへ戻る導線**: 複数アプリの移動が煩雑にならないよう、各アプリのヘッダ（`header`）には必ず「🏠 トップメニューへ戻る」ボタンかリンクを設置します。
- **モバイルファースト**: スーパーや外出先でのスマホ利用を第一に考えます。タップしやすい大きなボタン、10px以上の余白、一覧性の高さを最優先します。
- **絵文字の積極活用**: 視覚的に機能をすぐに把握できるよう、アイコンの代わりに標準の絵文字（＋, 🗑, 💾, 📷, 🛒, 💰 など）をボタンや見出しに取り入れます。
- **並び替え機能**: カテゴリ等の複雑な多層化は避け、1画面の平坦なリストに対して `Sortable.js` を用いた「ドラッグアンドドロップでの自由な配置換え」で使いやすさを担保します。

## 4. 文字入力・フォームへの配慮 (IME対応)
- 入力が即時確定して自動保存が走ると、日本語（IME）の漢字変換途中でエンターキーが効かなくなる等のバグのもとになります。
- 金額は即時反映でも良いですが、**テキスト入力（メモ、店名、商品名）の保存トリガーは `onchange` や `onblur`**（フォーカスが外れた時）に行うように設計してください。
